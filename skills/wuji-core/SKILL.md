## 核心机制

### 1. 真并发执行 (Hermes原生支持)

```
传统伪并发:  [专家A]→[专家B]→[专家C]  → 串行

Hermes真并发:
  [专家A]═══════
  [专家B]═══════  → 同时执行，真正的并行
  [专家C]═══════
```

**实现方式**:
- 每个专家对应独立的 `AgenticSession`
- 通过 `conversationId` 区分不同会话
- `agenticStore` 同时管理多个运行中的会话
- 多个专家真正同时执行任务

### 打靶场（沙盒）触发条件

以下情况必须进打靶场：
- 新Skill安装
- 新MCP配置
- 专家扩编/融合
- 重大配置变更
- 外部代码引入
- 权限变更

**测试内容**: 功能测试、安全测试、兼容性测试、性能测试

## 核心原则

1. **真并发执行** - 多专家同时运行，不是伪并行
2. **安全第一** - 5级权限管控，致命操作一票否决
3. **专业分工** - 每个专家只做自己领域的事
4. **流水线驱动** - 多专家协同，依赖清晰的并行执行
5. **动态扩编** - 没有就创建，缺了就补充
6. **矛与盾验证** - 实施者≠验证者，对抗性测试
