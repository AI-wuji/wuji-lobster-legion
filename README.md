## 运作机制

### 流水线驱动，禁止单兵作战

任何用户指令自动匹配 8 条作战流水线之一，多部门协同执行：

1. **软件产品全生命周期**（旗舰）— 情报侦察→产品规划→并行开发→质量关卡→加密发布→上线运营
2. **ComfyUI 插件开发** — 情报→开发→验证→发布
3. **安全攻防** — 防御加密 / 攻击逆向 / 安全评估
4. **小说创作** — 创作→包装→推广
5. **短剧/漫剧制作** — 策划→制作→发行
6. **数据情报** — 采集→分析→输出
7. **品牌建设** — 调研→设计→传播
8. **PPT 制作** — 素材→文案→设计→审核

### 阿极首尾闭环

所有流水线必须由阿极启动（参谋本部分析拆解）、由阿极收尾（汇总校验汇报）。专家不直接面对用户。

### 矛与盾验证机制

实施专家完成任务（矛）后，独立验证者（盾）以只读权限进行对抗性测试，出具 PASS / FAIL / PARTIAL 判定。实施者和验证者必须是不同的 Agent。

### 爆炸半径权限体系

| 级别 | 示例 | 处理方式 |
|------|------|---------|
| 无害 | 读取文件、搜索信息 | 自动放行 |
| 局部 | 写入单个文件 | 子 Agent 自主执行 |
| 中等 | 修改多个文件 | 需阿极确认 |
| 高危 | 删除数据、涉及金钱 | 必须用户确认 |
| 致命 | 不可逆操作 | 安全局一票否决 + 用户确认 |
